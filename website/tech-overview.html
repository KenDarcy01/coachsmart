FlutterFlow Frontend

Features

What CoachSmart does

A complete coaching management platform covering the full lifecycle of GAA team administration.

📅

Event Management

Create and manage matches, training sessions, and other events with rich metadata: location, opposition, audience restrictions by role level, meet times, and event codes. Events are scoped to teams within clubs.

✅

Attendance Tracking

Members respond to events with Accept/Decline/No Response. Coaches see real-time attendance summaries broken down by role and squad. Admins get detailed attendee lists with full member information.

🏆

Match Squad Builder

Generate match-day squads automatically from accepted attendees, grouped by position/squad. Export to CSV, Excel, or directly to a Google Sheet. Match reports and scores recorded post-game.

🚗

Car Pool Coordination

Members can offer or reserve seats in car pools for away events. Full visibility of available pools, seat counts, driver contact info, and reservation management per event.

🔔

Push Notifications

Real-time push notifications via Firebase FCM for attendance changes, event reminders, and team updates. Batched cron-based notification dispatch with a full in-app notification centre.

💳

Payments

Stripe-powered payment collection for events and memberships. Checkout session creation, webhook handling, and payment status tracking all handled via dedicated edge functions.

📧

Email Automation

Automated email reminders for unresponded events, attendance change alerts to admins, match report summaries, and a general-purpose send-email function for transactional messages.

👥

Team & Member Management

Multi-club, multi-team hierarchy. Members linked to users, assigned to squads, with role-based access levels (player, coach, manager, admin). Join-by-code onboarding for new members.

🔍

Search & Discovery

Event search with filters for date range, event type, code, opposition, and team. Admin-only features gated by role level to keep management capabilities separate from player views.

📊

Match Reports & Stats

Post-game match reports with score tracking by type, squad-level stats, and exportable reports. Structured match score detail tables for historical record keeping.

Role Grades

Roles are grouped into grades that determine the category of team member.

Grade 100Admin / Management
Grade 10Player / Athlete

Role Levels

Within each grade, role_level controls what a user can see. Higher level = more access. Events set a minimum audience level; members below it don't see the event.

Example: a match marked with audience level 20 (Senior) is hidden from Junior players at level 10.

Multi-Member User Support

A single user (login account) can be linked to multiple members via user_member_link. This supports the common GAA scenario where a parent manages accounts for themselves and a child, or a coach plays across multiple age groups.

Integrations

Third-party services

All integrations are handled through Supabase edge functions, keeping credentials server-side.

💳

Stripe — Payments

Full Stripe integration for event and membership payments. Checkout session creation, payment intent handling, webhook processing for payment confirmation, and transaction record storage. Both V1 and V2 checkout flows exist.

🔔

Firebase Cloud Messaging — Push Notifications

Real-time and scheduled push notifications delivered via FCM. Individual push via send-push-notification, webhook-triggered via trigger-push-notification, and batched cron delivery via cron_push_batch_notifications.

📊

Google Sheets — Squad Export

Export match-day squads directly to Google Sheets. OAuth token retrieval via get-google-token and spreadsheet creation/population via create_google_sheet_squads.

📧

Email — Transactional & Reminders

Automated emails for: event reminders to non-responders (event_email_reminder), match report summaries (event_match_report_email), attendance change alerts (send-change-of-attendance), and a general-purpose email sender (send-email).

⏱️

pg_cron — Scheduled Jobs

PostgreSQL-native cron scheduler for automated notification batching. The check_and_send_notifications function is called on schedule to dispatch pending notifications to the cron_batch_notification_send edge function.

🔐

Supabase Auth — User Management

JWT-based authentication. The handle_new_user trigger runs on signup to set up the user record and migrate any legacy account data. Auth tokens are passed with every API call from the FlutterFlow client.

Security

Security posture

Active hardening underway ahead of App Store submission. Core security model is Supabase RLS + SECURITY DEFINER functions with pinned search paths.

🔒

SECURITY DEFINER Functions

All 31 RPC functions run as the database owner with SET search_path = 'public' pinned, preventing schema hijacking attacks. Complex queries that require cross-table joins bypass RLS safely via these functions.

🛡️

Row Level Security

RLS enabled on Supabase tables. The is_owner_of_member_team_role function is used in RLS policies to verify team ownership before allowing write operations.

🔑

JWT Authentication

All client requests authenticated via Supabase JWT. The native iOS app sends auth tokens with every RPC call. PWA currently being updated to match (pending republish).

🏗️

Role-Based Access Control

Role levels control both data visibility (events with audience restrictions) and admin feature access. Grade-100 roles gate event creation, squad management, and admin panels.

Hardening Roadmap

Pre–App Store security plan

✅ Pin search_path on all SECURITY DEFINER functions

Applied via migration. All 31 functions now have SET search_path = 'public', closing the schema hijacking vector.

⏳ Enforce auth.uid() on user-scoped RPC functions

13 functions that accept a p_user_id parameter will be updated to ignore the parameter and use the authenticated caller's identity instead. Pending PWA republish.

⏳ Add team-membership guard to event-based functions

10 functions that return event data without checking team membership will gain an auth.uid() membership guard.

⏳ JWT validation on edge functions

9 high-risk edge functions that invoke service_role without verifying the caller's JWT will be updated to validate authentication first.

⏳ Remove unused DB views

30 of 31 database views are unused. Safe to drop — only view_match_squads is referenced in 3 export edge functions.
